# Инструкция для разработчика 1С: выгрузка заказов с товарами

Передай этот документ разработчику 1С, чтобы настроить выгрузку заказов с товарами в DriveControl.

---

## 1. URL и метод

**Метод:** `POST`  
**URL:** `https://mrdicoctfaxdrmoluqpi.supabase.co/functions/v1/orders-1c-ingest`

*(Если у вас другой проект Supabase — замените `mrdicoctfaxdrmoluqpi` на свой Project Ref из Dashboard.)*

**Заголовки:**
```
Content-Type: application/json
```

---

## 2. Обязательные поля

| Поле | Тип | Описание |
|------|-----|----------|
| `order_1c_id` | строка | Уникальный идентификатор заказа в 1С (номер, GUID и т.п.) |
| `delivery_address` | строка | Адрес доставки |

Без этих полей заказ не будет принят.

---

## 3. Поле «Товары» (`items`)

Чтобы в DriveControl отображались **товары заказа**, 1С должна передавать поле `items` в теле запроса.

### Вариант А — строка (рекомендуется)

Список товаров в одну строку, через запятую:

```json
{
  "order_1c_id": "ЗАК-001",
  "delivery_address": "г. Минск, ул. Примерная, д. 1",
  "items": "Молоко 2л, Хлеб белый, Яйца 10 шт."
}
```

Или через перенос строки:

```json
{
  "order_1c_id": "ЗАК-002",
  "delivery_address": "г. Минск, пр. Независимости, 45",
  "items": "Вода 5л\nСок яблочный 1л\nСыр 200г"
}
```

### Вариант Б — JSON (массив строк)

```json
{
  "order_1c_id": "ЗАК-003",
  "delivery_address": "г. Минск, ул. Немига, 12",
  "items": ["Молоко 2л", "Хлеб белый", "Яйца 10 шт."]
}
```

---

## 4. Остальные поля (по желанию)

| Поле | Тип | Описание |
|------|-----|----------|
| `order_date` | дата `YYYY-MM-DD` | Дата доставки. По умолчанию — сегодня |
| `customer_name` | строка | Имя клиента |
| `phone` | строка | Телефон |
| `delivery_time_slot` | строка | Временной слот (например "до 14", "9:00–12:00") |
| `amount` | число | Сумма заказа |

---

## 5. Формат запроса

### Один заказ

```json
{
  "order_1c_id": "ЗАК-001",
  "order_date": "2025-03-06",
  "customer_name": "Иванов И.И.",
  "delivery_address": "г. Минск, ул. Примерная, д. 1",
  "phone": "+375 29 123-45-67",
  "delivery_time_slot": "до 14",
  "items": "Молоко 2л, Хлеб белый, Яйца 10 шт.",
  "amount": 1850.50
}
```

### Несколько заказов (массив `orders`)

```json
{
  "orders": [
    {
      "order_1c_id": "ЗАК-001",
      "order_date": "2025-03-06",
      "customer_name": "Иванов И.И.",
      "delivery_address": "г. Минск, ул. Примерная, д. 1",
      "phone": "+375 29 123-45-67",
      "delivery_time_slot": "до 14",
      "items": "Молоко 2л, Хлеб белый, Яйца 10 шт.",
      "amount": 1850.50
    },
    {
      "order_1c_id": "ЗАК-002",
      "order_date": "2025-03-06",
      "customer_name": "Петров П.П.",
      "delivery_address": "г. Минск, пр. Независимости, 45",
      "phone": "+375 29 111-22-33",
      "delivery_time_slot": "до 18",
      "items": "Вода 5л, Сок яблочный 1л",
      "amount": 1200
    }
  ]
}
```

---

## 6. Поведение при повторной отправке

При повторной отправке заказа с тем же `order_1c_id`:
- запись **обновляется** (upsert);
- статус и привязка к водителю **не перезаписываются** — их задаёт кабинет DriveControl;
- поля `items`, `amount`, адрес и т.п. **обновляются** из 1С.

Можно подгружать заказы по мере формирования и обновлять их в течение дня.

---

## 7. Проверка

1. Отправь тестовый POST с `items` через Postman или из 1С.
2. Открой DriveControl → вкладка «Заказы из 1С» → выбери нужную дату → «Обновить».
3. В таблице заказов должны отображаться товары в колонке «Товары».

---

## Кратко для 1С

1. При формировании заказов на день вызывать `POST https://mrdicoctfaxdrmoluqpi.supabase.co/functions/v1/orders-1c-ingest`
2. Обязательные поля: `order_1c_id`, `delivery_address`
3. Для выгрузки товаров — передавать поле `items` (строка или массив строк)
4. Формат JSON — см. примеры выше
