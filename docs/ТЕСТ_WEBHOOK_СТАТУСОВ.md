# Тест отправки статусов заказов (webhook.site)

Пошаговая инструкция, как проверить, что статусы заказов уходят из DriveControl при действиях водителя.

---

## Шаг 1. Получить тестовый URL от webhook.site

1. Открой в браузере: **https://webhook.site**
2. Страница сразу покажет тебе **уникальный URL** — он выделен вверху.
   - Пример: `https://webhook.site/a1b2c3d4-e5f6-7890-abcd-ef1234567890`
3. **Скопируй этот URL** целиком (Ctrl+C).
4. Оставь вкладку webhook.site открытой — на ней будут видны все входящие запросы.

---

## Шаг 2. Установить URL в Supabase (вариант А — через Dashboard)

1. Зайди на **https://supabase.com/dashboard**
2. Выбери свой проект (DriveControl).
3. В левом меню открой **Edge Functions** (значок молнии).
4. Вверху перейди на вкладку **Secrets** (или настройки функции).
   
   Или: **Project Settings** (шестерёнка внизу слева) → **Edge Functions** → раздел **Secrets**.

5. В блоке **Add new secret** заполни:
   - **Key (ключ):** `ONE_C_WEBHOOK_URL`
   - **Value (значение):** вставь скопированный URL с webhook.site, например:
     ```
     https://webhook.site/a1b2c3d4-e5f6-7890-abcd-ef1234567890
     ```
6. Нажми **Save** / **Add secret**.

Секрет применится сразу, перезапуск функций не нужен.

---

## Шаг 2. Установить URL в Supabase (вариант Б — через терминал)

1. Открой терминал в папке проекта: `c:\drivecontrol`
2. Убедись, что проект привязан:
   ```bash
   supabase link --project-ref mrdicoctfaxdrmoluqpi
   ```
   *(если уже привязан — будет сообщение об этом)*

3. Установи секрет (подставь свой URL с webhook.site):
   ```bash
   supabase secrets set ONE_C_WEBHOOK_URL=https://webhook.site/твой-uuid-сюда
   ```
   Пример:
   ```bash
   supabase secrets set ONE_C_WEBHOOK_URL=https://webhook.site/a1b2c3d4-e5f6-7890-abcd-ef1234567890
   ```

4. Проверь, что секрет сохранён:
   ```bash
   supabase secrets list
   ```
   В списке должна быть строка `ONE_C_WEBHOOK_URL`.

---

## Шаг 3. Проверить, что функция задеплоена

Функция `push-order-status-to-1c` должна быть в Supabase. Если не уверен:

```bash
supabase functions deploy push-order-status-to-1c --no-verify-jwt
```

---

## Шаг 4. Тест в приложении

1. Открой DriveControl (вкладка **Водители**).
2. Войди под ролью водителя (или выбери тестового водителя).
3. Открой маршрут, в котором есть заказ из 1С (с кнопками «В доставке», «Доставлен», «Отменён»).
4. Нажми **«В доставке»** или **«Доставлен»** для любого заказа 1С.

---

## Шаг 5. Смотреть результат на webhook.site

1. Вернись на вкладку **webhook.site**.
2. Должен появиться **новый запрос** (вверху списка).
3. Кликни по нему — откроется детали:
   - **Request URL** — твой URL
   - **Request method** — POST
   - **Request body** — JSON примерно такого вида:
     ```json
     {
       "order_1c_id": "ЗАК-001",
       "status": "delivered"
     }
     ```

Если запрос виден — статусы из DriveControl успешно отправляются.

---

## Проверка без приложения (если ничего не приходит)

Чтобы понять, где обрыв — в приложении или в Supabase — вызови функцию **напрямую**:

1. Открой **Postman** (или вкладку в браузере с расширением для API).
2. Создай запрос:
   - **Метод:** POST
   - **URL:** `https://mrdicoctfaxdrmoluqpi.supabase.co/functions/v1/push-order-status-to-1c`
   - **Headers:**
     - `Content-Type: application/json`
     - `Authorization: Bearer <anon_key>` — ключ из Supabase → Settings → API → anon public, или из `js/config.js`
   - **Body** (raw, JSON):
     ```json
     {"order_1c_id": "TEST-001", "status": "delivered"}
     ```
3. Нажми **Send**.
4. Сразу открой webhook.site — должен появиться запрос.

**Результат:**
- **Запрос появился на webhook.site** → секрет и функция настроены верно. Проблема в приложении: в маршруте нет заказов 1С с `order_1c_id`, или ты жмёшь не на ту кнопку.
- **Запроса нет** → проверь: `supabase secrets list` (есть ли `ONE_C_WEBHOOK_URL`?), задеплоена ли функция, правильный ли URL webhook.site.

---

## Если запросов нет (чековый список)

1. **Проверка цепочки:**  
   Вызови функцию из Postman (см. выше). Появился запрос на webhook.site?

2. **Проверь секрет:**  
   `supabase secrets list` — есть ли `ONE_C_WEBHOOK_URL`?  
   Если нет: `supabase secrets set ONE_C_WEBHOOK_URL=https://webhook.site/твой-uuid`

3. **Проверь заказ в маршруте:**  
   Кнопки «В доставке» / «Доставлен» / «Отменён» видны только у точек с подписью **«Заказ 1С: …»**. У обычных адресов, поставщиков или партнёров — другие кнопки, webhook не вызывается.

4. **Есть ли заказы 1С в маршруте?**  
   Маршрут должен содержать точки, созданные из вкладки «Заказы из 1С» (через «Перенести на карту» → распределение → «Отправить в путевые листы»). Только у таких точек есть `order_1c_id`.

5. **Консоль браузера (F12) → Network:**  
   При нажатии «Доставлен» должен уйти запрос на `.../functions/v1/push-order-status-to-1c`. Если запроса нет — точка не считается заказом 1С.

6. **Логи в Supabase:**  
   Edge Functions → `push-order-status-to-1c` → Logs. Там будут ошибки, если функция падает.

---

## Когда тест закончен

Чтобы не слать лишние запросы на webhook.site, можно:

- Удалить секрет в Dashboard (Secrets → удалить `ONE_C_WEBHOOK_URL`),  
или
- Заменить его на реальный URL 1С, когда он будет готов.
