# Миграция legacy → единое React-приложение (выполнено)

Приложение переведено на **единую точку входа** — React SPA с роутером. Старый интерфейс сохранён в `index.legacy.html` для справки.

---

## Шаги

### 1. Точка входа и роутер
- Сделать **единственную** точку входа для продакшена: `index.html` с `<div id="root">` и скрипт Vite (`src/main.jsx`).
- Подключить React Router. Маршруты: `/`, `/suppliers`, `/partners`, `/drivers`, `/vehicles`, `/distribution`, `/schedule` и т.д.
- Сборка: `npm run build` → выход в `dist/`. Netlify должен отдавать SPA из `dist/` (или корня с `dist` как publish directory).

### 2. Перенос разделов по одному
- **Распределение** — уже есть (`src/pages/distribution`). Подключить как маршрут `/distribution`.
- **Поставщики** — перенести логику из `js/app.js` + `js/db.js` в React: страница, список, модалка, карта (если есть), Supabase API.
- **Партнёры** — то же, из `js/db-partners.js` и соответствующего UI.
- **Водители** — из `js/vehicles.js` (getAllDrivers и т.д.) + модалка водителя, кнопка «Распределённые», «График смен».
- **Автомобили** — раздел машин, пробег, ТО, история; по мере переноса подключать к Supabase.
- **График смен** — таблица из `js/` (schedule), сохранение в БД.

Каждый раздел: страница в `src/pages/<section>`, общие сущности в `src/entities`, API в `src/shared/api` или в сущностях.

### 3. Общая оболочка
- Общий layout: шапка с навигацией (как текущие вкладки), контент по маршруту.
- Supabase: один клиент (`src/shared/api/supabaseClient.js`), при необходимости те же ключи из env (Vite) или `window.SUPABASE_CONFIG` для обратной совместимости.

### 4. Отказ от legacy
- Когда все разделы перенесены, старый `index.html` и папка `js/` (кроме возможно статики) не используются.
- Netlify: build command `npm run build`, publish directory `dist`.

---

## Порядок работ (рекомендуемый)
1. Роутер + layout + маршрут только на «Распределение» (уже в React).
2. Переключить деплой на Vite build, проверить, что в проде открывается React и «Распределение».
3. По одному добавлять разделы: Поставщики → Партнёры → Водители → Автомобили → График смен.

Миграцию можно делать пошагово, не ломая текущий прод: например, сначала отдавать React только по `/distribution`, остальное оставить на старом HTML до переноса всех разделов.
